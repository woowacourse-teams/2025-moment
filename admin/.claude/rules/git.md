# Git 규칙

## 브랜치 네이밍

```
<type>/<이슈번호>-<짧은설명>
```

| 타입 | 사용 시점 |
|------|----------|
| `feat/` | 새로운 기능 |
| `fix/` | 버그 수정 |
| `refactor/` | 코드 구조 개선 |
| `chore/` | 빌드, 설정, 의존성 변경 |

예시: `feat/#123-user-list-page`

## 커밋 메시지

형식: `<type>: <제목>`

| 타입 | 사용 시점 |
|------|----------|
| `feat` | 새로운 기능 |
| `fix` | 버그 수정 |
| `refactor` | 코드 구조 개선 |
| `style` | 포맷팅만 변경 |
| `docs` | 문서 작업 |
| `test` | 테스트 작업 |
| `chore` | 빌드/설정 변경 |

예시: `feat: 사용자 목록 페이지네이션 추가`

## 규칙

- 원자적 커밋 (논리적 변경 단위 하나당 커밋 하나)
- main/master에 `--force` push 금지
- `--no-verify` 사용 금지 (명시적 허가 없이)
